## やりたかったこと

「今年はパブリックに学ぶ」というテーマで、Obsidianボルトにある学習記録を整理・推敲してブログとして公開したかった。

理想のワークフロー：
1. Obsidianで日々の学習を記録
2. AIと一緒に推敲して読みやすくする
3. GitHub Pagesで公開

## Claude Codeとの協業

VS CodeのClaude Code拡張を使って、対話しながらシステムを構築した。

### 作ったもの

**1. Python CLIツール (`lip`)**
- `lip scan` - ボルトから公開候補のノートをスキャン
- `lip review` - プライバシーチェック（電話番号、住所、パスワードなど検出）
- `lip convert` - Obsidian形式からHugo形式に変換

**2. Hugoブログ**
- これはローカルにHugoサーバというものを立てて、ObsidianのMDファイルををHTMLに動的に変換してくれるものらしい。
- PaperModテーマ
- 日本語対応
- GitHub Actions でのデプロイ設定
- 

**3. CLAUDE.md**
- 推敲の方針やプロジェクト情報を記載
- Claude Codeが毎回読み込んで、一貫した対応ができる
- だんだんと自分の個性が出るようにCLAUDE.mdの内容を充実させていきたい

### ハイブリッド方式

最初は「完全自動化」を考えていたが、推敲部分はAIとの対話でやる「ハイブリッド方式」に落ち着いた。

- **自動化**: スキャン、プライバシーチェック、変換、公開
- **対話**: ノートの推敲（Claude Codeと会話しながら）

APIコストもかからないし、対話しながらの方が細かいニュアンスを伝えられる。

### 工夫したポイント

**ブログ記事の置き場所**

最初はHugoプロジェクト内に記事を置いていたが、Obsidianボルト内の `Public/` フォルダに変更した。

```
/Users/shiori/Desktop/KM/
├── Public/              ← ブログ記事
│   └── posts/
├── 05_Inbox/            ← 学習ノート
└── ...
```

これでObsidian内で記事を編集でき、他のノートからリンクも貼れる。

**プライバシー検出の調整**

最初はTikTokのURLやタイムスタンプが「電話番号」として誤検出（偽陽性）されていた。
パターンを調整して、URLは除外するようにした。
本当は、プライバシーに配慮してっていうのは、電話番号とかパスワードもそうだけど、あまりにもプライベートな話とか赤裸々な話？を除くみたいな意味合いだったんだけど、まあ、それは長期的な目標にしようと思う。

## 感想
